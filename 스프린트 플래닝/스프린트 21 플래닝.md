### **1. 운동 파트너를 찾는 유저로서 성별, 나이, 종목 등 구체적인 조건으로 필터링을 하고 싶다. 그래서 지도에서 나와 조건이 맞지 않는 사람을 일일이 확인하는 수고를 덜고, 딱 맞는 후보군만 볼 수 있다. (10)**

- 개발 및 학습 목록
    1. **사용자 검색 (필터링 적용)** 
    - Backend
        - GET /api/v1/members
            - 화면 기준 viewPort 범위 지도 좌표 받아서 해당 범위 내의 회원 좌표 검색
                - 왼쪽위, 오른쪽 아래 좌표
        - **Response:** 사용자 리스트 (사용자 운동장소 좌표 포함)
    - 클라이언트
        - 좌표 이동에 따라 api 재호출할지 결정
        - 쓰로틀링
        - 필터에 맞게 캐싱한 데이터에서 타회원 마커 표시
            - gender: (String) M/F/ALL (U 포함)
            - ageRange: (String) 20s, 30s...
            - preferredExercise: (List) [Soccer, Gym...]
            - daysOfWeek: (List) [Mon, Wed, Fri] (다중선택)
            - skillLevel: (String) Beginner/Intermediate/Expert … 6단계
            - exerciseScore: 0~100 (20점 이상, 40점 이상, 60점 이상, 80점 이상)
    - 이전 스프린트 미비작업
        - 백엔드 : 마이페이지 조회시 삭제된 선호운동이 보이는 이슈 해결
        - 안드로이드 : 선호운동 API 연결
        - 프론트엔드 : 웹소켓 연결
